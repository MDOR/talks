What is React?

React is a JS library to build UI components.

React is recommended to build large application with data that changes over the time, React will manage all the UI updates.

JSX

JSX is an optional XML-like syntax extension to ECMAScript that can be used to define a HTML tree structure in a JavaScript file.

The JSX expressions will be transformed to JavaScript.

/*Syntax React*/
var Component = React.createClass('p', {className: 'my-component'}, 'texto')

/*Syntax JSX*/
var Component = React.createClass({
                  render : function () {
                    return (<p className="my-component"> texto </p>)
                  }
                })

React concepts

Props

Provide configuration values for the component, these values aren't mutable (read only).

States

States are used to store information about the component that can change over the time.

Vitual Dom

Fake DOM stored in memory to check any change into the component.

States vs Props

  • Both are plain objects
  • Both can have default values
  • Both are accessed via this

Props

  • Props are passed into a component from above, either a parent component or from the starting scope where React is originally rendered.
  • Props are intended as configuration values passed into the component.
  • Props are immutable to the component receiving them.

States

  • State should always start with a default value and then the state is mutated internally by the component using setState.
  • I'ts private.
  • It should contain just the information to handle changes into the UI.

Events

Events can be added to DOM nodes, React can handle events for:

  • Clipboard
  • Keyboard
  • Focus
  • Form
  • Mouse
  • Touch
  • Image

Also we can use stopPropagation and preventDefault.

Lyfecycle Methods

  • getInitialState: Return value is the initial value for state.
  • getDefaultProps: Sets fallback props values if props aren’t supplied.
  • componentWillMount: Invoked once, on both client & server before rendering occurs.
  • componentDidMount: Invoked once, only on the client, after rendering occurs.
  • shouldComponentUpdate: Return value determines whether component should update.
  • componentWillUnmount: Invoked prior to unmounting component.
